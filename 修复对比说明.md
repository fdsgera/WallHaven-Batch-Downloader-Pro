# 修复对比说明

## 核心修复点对比

### 修复点 1：元数据引用管理

#### ❌ 修复前（问题代码）
```python
class WallhavenGUI:
    def __init__(self):
        # 没有独立的元数据引用
        self.current_preview_index = 0
        self.preview_cache = {}
        # self.metadata = None  # 缺失！
    
    def show_current_image(self):
        # 直接使用 downloader 的元数据
        metadata = self.downloader.metadata  # 问题：引用不稳定
        if not metadata:
            return
        
        current_meta = metadata[self.current_preview_index]
        # 使用元数据中的索引
        index = current_meta['index']  # 问题：不是预览索引
```

#### ✅ 修复后（正确代码）
```python
class WallhavenGUI:
    def __init__(self):
        # 添加独立的元数据引用
        self.current_preview_index = 0
        self.preview_cache = {}
        self.metadata = None  # 新增：独立引用
    
    def enable_preview(self):
        def _enable():
            # 保存元数据的独立副本
            if self.downloader and self.downloader.metadata:
                self.metadata = self.downloader.metadata  # 保存引用
                self.current_preview_index = 0
                self.preview_cache = {}  # 清空缓存
    
    def show_current_image(self):
        # 使用保存的独立引用
        metadata = self.metadata  # 修复：使用稳定的引用
        if not metadata:
            return
        
        current_meta = metadata[self.current_preview_index]
        # 使用实际的预览索引
        index = self.current_preview_index + 1  # 修复：正确的索引
        total = len(metadata)
```

**关键差异**：
- ❌ 旧版：直接引用 `self.downloader.metadata`，当下载器重新创建时引用失效
- ✅ 新版：保存独立的 `self.metadata` 引用，不受下载器生命周期影响

### 修复点 2：预览索引计算

#### ❌ 修复前
```python
def show_current_image(self):
    current_meta = metadata[self.current_preview_index]
    # 错误：使用元数据中存储的下载索引
    index = current_meta['index']  # 这是下载序号，不是预览位置
    total = len(metadata)
    
    self.preview_label.configure(
        image=photo, 
        text=f"正在预览：第 {index}/{total} 张"
    )
```

**问题**：`current_meta['index']` 是图片在下载队列中的序号（1, 2, 3...），但用户可能从中间开始浏览，导致显示的序号与实际位置不符。

#### ✅ 修复后
```python
def show_current_image(self):
    # 正确：使用当前预览索引
    index = self.current_preview_index + 1  # 从0开始，显示时+1
    total = len(metadata)
    
    self.preview_label.configure(
        image=photo, 
        text=f"正在预览：第 {index}/{total} 张"
    )
```

**改进**：直接使用 `self.current_preview_index` 计算，确保显示的序号与用户浏览位置一致。

### 修复点 3：状态重置

#### ❌ 修复前
```python
def start_download(self):
    # 清空日志和预览
    self.log_text.delete('1.0', 'end')
    self.preview_label.configure(image=None, text="等待元数据收集...")
    self.current_preview_index = 0
    # self.preview_cache = {}  # 可能缺失
    # self.metadata = None  # 缺失！
```

**问题**：
- 缓存可能没有清空，导致显示旧图片
- 元数据引用没有重置，可能指向旧数据

#### ✅ 修复后
```python
def start_download(self):
    # 清空日志和预览
    self.log_text.delete('1.0', 'end')
    self.preview_label.configure(image=None, text="等待元数据收集...")
    self.current_preview_index = 0
    self.preview_cache = {}  # 确保清空缓存
    self.metadata = None  # 重置元数据引用
    
    # 禁用预览按钮
    self.prev_btn.configure(state="disabled")
    self.next_btn.configure(state="disabled")
```

**改进**：完整重置所有预览相关状态，确保每次下载都是全新开始。

## 问题场景重现

### 场景 1：连续下载不同关键词

#### ❌ 修复前的问题
```
1. 用户搜索 "anime"，下载 10 张图片
2. 预览显示 anime 图片（正常）
3. 用户搜索 "landscape"，下载 10 张新图片
4. 预览仍然显示 anime 图片（BUG！）
```

**原因**：`self.downloader.metadata` 指向新的下载器，但预览可能还在使用旧的引用或缓存。

#### ✅ 修复后的行为
```
1. 用户搜索 "anime"，下载 10 张图片
2. 预览显示 anime 图片（正常）
3. 用户搜索 "landscape"，开始新下载
   - self.metadata = None（重置）
   - self.preview_cache = {}（清空）
4. 元数据收集完成后
   - self.metadata = 新的元数据（更新）
5. 预览显示 landscape 图片（正确！）
```

### 场景 2：预览序号错乱

#### ❌ 修复前的问题
```
元数据：[
    {'index': 1, 'url': '...'},  # 下载序号 1
    {'index': 2, 'url': '...'},  # 下载序号 2
    {'index': 3, 'url': '...'},  # 下载序号 3
]

用户浏览：
- 点击"下一张" → 显示 "第 2/3 张"（正确）
- 点击"下一张" → 显示 "第 3/3 张"（正确）
- 点击"上一张" → 显示 "第 2/3 张"（正确）

但如果元数据的 index 不连续：
元数据：[
    {'index': 5, 'url': '...'},  # 下载序号 5
    {'index': 8, 'url': '...'},  # 下载序号 8
    {'index': 12, 'url': '...'},  # 下载序号 12
]

用户浏览：
- 第一张 → 显示 "第 5/3 张"（错误！）
- 第二张 → 显示 "第 8/3 张"（错误！）
```

#### ✅ 修复后的行为
```
无论元数据的 index 值如何，都使用预览索引：
- 第一张 → 显示 "第 1/3 张"（正确）
- 第二张 → 显示 "第 2/3 张"（正确）
- 第三张 → 显示 "第 3/3 张"（正确）
```

## 代码质量改进

### 1. 线程安全
```python
def enable_preview(self):
    """启用预览功能（元数据收集完成后调用）"""
    def _enable():
        # 所有 GUI 操作都在主线程中执行
        if self.downloader and self.downloader.metadata:
            self.metadata = self.downloader.metadata
            # ...
    
    self.root.after(0, _enable)  # 确保在主线程执行
```

```python
self.metadata = None              # 元数据引用

```
