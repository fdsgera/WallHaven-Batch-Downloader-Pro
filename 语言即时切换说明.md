# 语言切换即时生效更新说明

## 问题描述
之前的版本中，切换语言后点击"应用"按钮，界面文本不会立即更新，需要重启程序才能看到新语言。

## 解决方案

### 1. 添加UI控件引用字典
在 `__init__` 方法中添加了 `self.ui_widgets = {}` 字典，用于保存所有需要动态更新的UI控件引用。

### 2. 保存UI控件引用
在 `build_ui()` 方法中，将所有需要更新文本的控件保存到 `ui_widgets` 字典中：
- 标题标签
- 设置按钮
- 所有输入区域的标签
- 所有复选框
- 所有按钮
- 日志和预览区域的标签

### 3. 实现动态更新
重写了 `update_ui_language()` 方法，使其能够：
- 更新窗口标题
- 遍历所有保存的UI控件
- 使用 `configure(text=...)` 方法更新每个控件的文本
- 立即应用新语言，无需重启

### 4. 更新的UI元素
以下界面元素现在可以即时切换语言：
- ✅ 窗口标题
- ✅ 设置按钮
- ✅ 所有标签（关键词、数量、路径等）
- ✅ 浏览按钮
- ✅ 分类复选框（常规、动漫、人物）
- ✅ 分级复选框（安全、擦边、限制）
- ✅ API Key标签
- ✅ 保存配置按钮
- ✅ 开始/停止下载按钮
- ✅ 日志和预览标题
- ✅ 预览控制按钮（上一张、刷新、下一张）
- ✅ 预览状态文本

### 5. 限制说明
由于 CustomTkinter 的技术限制，以下元素无法动态更新（需要重启程序）：
- ⚠️ 输入框的占位符文本（placeholder_text）

## 使用效果

现在用户可以：
1. 打开设置窗口
2. 选择新语言
3. 点击"应用"按钮
4. **立即看到界面语言切换**，无需重启程序

## 技术实现

### 代码示例

```python
# 保存UI控件引用
self.ui_widgets['title_label'] = ctk.CTkLabel(...)
self.ui_widgets['settings_btn'] = ctk.CTkButton(...)

# 动态更新文本
def update_ui_language(self):
    self.root.title(self.lang['title'])
    self.ui_widgets['title_label'].configure(text=self.lang['title'])
    self.ui_widgets['settings_btn'].configure(text=self.lang['settings'])
    # ... 更新其他控件
```

## 测试步骤

1. 启动程序（默认英文界面）
2. 点击右上角 "⚙️ Settings" 按钮
3. 选择 "简体中文"
4. 点击 "Apply" 按钮
5. **观察界面立即切换为中文**
6. 再次打开设置，切换回 "English"
7. 点击 "应用"
8. **观察界面立即切换为英文**
